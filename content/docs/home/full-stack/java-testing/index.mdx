## Summary of Popular Java Testing Frameworks

| Category | Framework | Notes |
|----------|-----------|-------|
| **Unit Testing** | **JUnit 5** | Core testing foundation. Default choice for all Java projects. |
| | **TestNG** | Alternative to JUnit; strong in parallel test execution. |
| **Mocking** | **Mockito** | Works standalone; often used with JUnit. Plays nicely in Spring Boot too. |
| | **EasyMock** | Older not widely used anymore but still independent. |
| | **JMockit** | Can mock static/final/private; works without Spring. |
| **Integration / System Testing** | **REST Assured** | Standalone API testing library. Spring Boot only makes setup easier. |
| | **TestContainers** | Independent; spins up Docker containers. Useful with or without Spring. |
| | **Spring Boot Test** | Tightly coupled to Spring ðŸŒ±. Provides annotations like `@SpringBootTest`, `@DataJpaTest`. |
| **BDD (Behavior-Driven Dev)** | **Cucumber (Gherkin)** | Works in plain Java projects and Spring Boot. Spring just helps with DI glue. |
| | **JBehave** | Older BDD option; independent. |
| **Architecture / Advanced Testing** | **JUnit QuickCheck** | Property-based testing; independent. |
| | **ArchUnit** | Tests architectural rules; independent of Spring. |
| **UI / E2E Testing** | **Selenium / Selenide** | Fully independent; Spring doesnâ€™t matter here. |

## âœ… Best Practices
  - Write small, independent tests
